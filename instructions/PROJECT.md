# 仕事の基礎ガイド - AI支援指示書

このプロジェクトは新人・若手社員向けの「仕事の基礎」を学ぶためのガイドドキュメントです。

## AI指示書システムの使用

タスク開始時は`instructions/ai_instruction_kits/instructions/ja/system/ROOT_INSTRUCTION.md`を読み込んでください。

## プロジェクト設定

- **目的**: 仕事の基礎ガイドの作成・改善
- **対象読者**: 新人・若手社員（入社1〜3年目）
- **言語**: 日本語
- **出力形式**: HTML, PDF
- **チェックポイント管理**: 有効
- **チェックポイントスクリプト**: scripts/checkpoint.sh

## 対象読者の特徴

- 入社1〜3年目の若手社員
- 「何をすればいいか分からない」「何をどう報告すればいいか分からない」と感じている
- 指示待ちになりがち、一人で抱え込みやすい
- 報告・相談・タスク管理・時間管理に課題がある

## ガイドの目的

**受動的（reactive）→ 主体的（proactive）への成長を支援する**

具体的には：
- 「指示がないと動けない」→「自分で考えて動ける」
- 「報告の仕方が分からない」→「適切に報告・相談できる」

## 重要なパス

- AI指示書システム: `instructions/ai_instruction_kits/`
- ガイド本文: `reports/work-guide.qmd`
- 図表: `sources/diagrams/`
- 出力先: `output/`
- チェックポイントスクリプト: `scripts/checkpoint.sh`

## ガイドの構成

### Part 1: 日常業務編
- 朝イチでやること（毎日のルーティン）
- タスク管理の基本ルール（4象限マトリクス、「やらない」の判断基準）
- 報告の仕方（基本構造、場面別テンプレート、良い例・悪い例）
- 時間を区切って作業する（30分ルール、タイムボックス、タイムブロック）

### Part 2: プロジェクト遂行編
- プロジェクト参加時の心構え（作業計画書、分からないことリスト）
- タスク遂行のフロー
- 会議の参加の仕方（参加者として、ファシリテーターとして）
- 成果物作成のポイント

### Part 3: 困ったときの対処法
- よくある困りごとと解決策
- 上司との1on1の活用

### 付録: チェックリスト
- 毎日・週次・月次・プロジェクト開始時

## 編集時の注意点

### 文体・トーン
- 上から目線ではなく、**寄り添う**トーンで
- 「〜すべき」より「〜するとよい」「〜がおすすめ」
- 具体的な例を多く入れる
- NGパターンとOKパターンを対比で示す

### 図表の作成
- フローチャートやマトリクスはSVG形式で作成
- シンプルで分かりやすいデザイン
- 日本語テキストを含む
- `sources/diagrams/` に配置

## 重要なキーワード・概念

| 概念 | 説明 |
|------|------|
| 自律的 | 指示を待たずに自分で考えて動ける |
| 30分ルール | 30分詰まったら相談する |
| 4象限マトリクス | 緊急度×重要度でタスクを分類 |
| タイムボックス | 時間を区切って集中する |
| 結論から話す | 報告の基本構造 |
| 60%で見せる | 完璧を目指さず早めに共有 |
| 属人化を避ける | 「自分しかできない」は危険信号 |

## ビルドコマンド

```bash

# HTMLのみ
make report-html

# PDFのみ
make report-pdf

# プレビュー
make preview
```
